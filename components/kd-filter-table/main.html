插槽自定义控件如何编写 ​

  • formItemType:slot
  • filterXxx
  • modalValue,updateModelValue
js
1. formItemType设置为slot
2. 在组件内部使用name为filterXx的插槽，为了区分其他插槽，均以filter+字段首字母大写绑定
3. 需要绑定控件:model-value及change|input等等事件 相当于重写v-model语法糖 为啥不直接绑定v-model，因为走插槽上来的数据不是响应式的，因此只有重实现v-model，这样Filter内部才会获取到当前控件值，请确保执行上述操作

{
    formItemType: "slot",
    formItemLabel: "我是slot控件",
    formItemKey: "field",
},

<kd-filter-table>
    <template #filterField="{ modelValue, updateModelValue }">
    	<el-switch :model-value="modelValue.value" @change="updateModelValue" />
    </template>
</kd-filter-table>

Filter属性 ​

js
export type TFilterProps = {
  // filter
  filterName?: string; // 过滤器名称唯一标识 必传 不传时默认取当前router的name 当多组使用时 需要传入用于唯一标识
  filterList?: TFilterItem[]; // 过滤器表单集合
  filterCol?: {
    xs?: number;
    sm?: number;
    md?: number;
    lg?: number;
    xl?: number;
  }; // 自定义过滤器的布局 参看EMP layout
  filterFieldsColon?: boolean; // 是否搜索字段中文后面跟“：”
  filterQueryCache?: boolean; // 是否需要缓存过滤器值 TODO: 暂无使用
  filterDefaultBtn?: boolean; // 是否默认操作按钮 搜索 重置
  filterCache?: boolean; // 是否缓存当前针对filter的调整 TODO: 暂无实现

  filterEnableTools?: boolean; // 是否开启工具按钮
};

export type TFilterItem = {
  filterKey: string | number; // 唯一id
  key?: string; // 由formItemLabel+formItemKey组成唯一
  // common
  formItemType?: string; // 表单项类型 目前支持input select datePicker slot
  formItemLabel?: string; // 表单项标题
  formItemRules?: string; // 表单项校验
  formItemHidden?: boolean | Function; // 表单项隐藏
  formItemKey?: string; // 表单项key
  formItemInitValue?: any; // 控件初始默认值 根据所属控件设置对应控件类型值
  formItemRequired?: boolean; // 表单项 是否必填
  // 被用到的公共属性
  placeholder?: string;
  disabled?: boolean;
  // select业务属性
  selectOptions?: any[];
  selectKeyValueField?: { value: string; label: string | Function };
  // datePicker业务属性
  datePickerReallyKeys?: string; // 针对daterange处理分别绑定真实的key 用逗号分割 用于http query
  // 公共事件
  click?: Function;
  blur?: Function;
  focus?: Function;
  clear?: Function;
  change?: Function;
  input?: Function;
  visibleChange?: Function;
  // select特殊事件
  removeTag?: Function;
  // datePicker特殊事件
  calendarChange?: Function;
  panelChange?: Function;
};

Table表格 ​

  • 一般情况下，无需关心分页器事件内部会自己处理，只需要绑定table-request-method属性、表格字段即可，其余table属性均走官网来设置组件自身上

🌮支持模板构造column ​

  • 模板模式只是为了兼容之前系统的老代码不想大规模改动

  • 开发环境性能不如props模式，因为会将编译模板后的vnode再次进行动态创造
