el-upload包装 ​

  • 主要解决，通常中后台业务中，文件通常集成了oss文件服务器，而提交的业务数据按字符串拼接存储，因此提炼出了常见的业务场景，比如表单中上传单个附件、上传图片、上传视频、多文件列表上传等等
  • 为了业务通用性，提炼出比较公共的部分作为该组件的配置项进行初始化配置，具体可见配置项章节
  • 针对常见的上传操作，也封装了比较通用的hooks，具体可见hooks章节

  uploadType?: TUploadType; // input：带input image：大小按身份证大小初始 video: 大小按身份证大小初始 list: 列表形式(非原生)

4种模式下公共属性 ​

typescript
interface IcommonParams {
  modelValueString: string; // 拼接upload string 可直接提交后端
  modelValueFiles?: UploadFiles[]; // 获取附件原始信息 只会在新上传操作下有值，编辑时由于走的业务字符串则无详细信息
}

export interface ICommonProps extends IcommonParams {
  // 需要hooks pramas参数
  limitSize?: number; // 单文件大小 M 为单位
  screenFullLoading?: boolean; // 是否全屏loading效果
  loadingType?: "loading" | "progress"; // loading:按钮v-loading状态 progress 按钮0-100变化
  btnLabel?: string; // 上传按钮名称
  showDownload?: boolean; // 是否显示下载按钮
  showDelete?: boolean; // 是否显示删除按钮
  link?: boolean; // 按钮link状态 且变为“上传”

  onSuccessCb?: Function;
  onErrorCb?: Function;
  onProgressCb?: Function;
  onChangeCb?: Function;
  onBeforeUploadCb?: Function;
}

透传了原生el-upload属性 ​

你可以在组件上按el-upload属性来设置，但是不建议覆盖kd-upload已声明的事件

使用 ​

list
input
image

上传图片

video

上传视频


启用
禁用
<template>
  <el-form :model="form" label-width="120px" :disabled="form.disabled">
    <el-form-item label="list">
      <kd-upload
        v-model:modelValueString="form.file1"
        v-model:modelValueFiles="form.files1"
        upload-type="list"></kd-upload>
    </el-form-item>
    <el-form-item label="input">
      <kd-upload
        v-model:modelValueString="form.file2"
        v-model:modelValueFiles="form.files2"></kd-upload>
    </el-form-item>
    <el-form-item label="image">
      <kd-upload
        v-model:modelValueString="form.file3"
        v-model:modelValueFiles="form.files3"
        upload-type="image"></kd-upload>
    </el-form-item>
    <el-form-item label="video">
      <kd-upload
        v-model:modelValueString="form.file4"
        v-model:modelValueFiles="form.files4"
        upload-type="video"
        btn-label="上传视频"></kd-upload>
    </el-form-item>
  </el-form>
  <el-button size="small" @click="onSubmit">获取值</el-button>
  <br />
  <el-switch v-model="form.disabled" active-text="禁用" inactive-text="启用" />
</template>

<script setup lang="ts">
// ---------------------------------------- Import Business --------------------------------------------- //
// ---------------------------------------- Import Vue & Use Router ------------------------------------- //
import { reactive, onMounted } from "vue";
// ---------------------------------------- Reactive Data & Other --------------------------------------- //
const form = reactive({
  file1: "",
  files1: [],
  file2: "",
  files2: [],
  file3: "",
  files3: [],
  file4: "",
  files4: [],
  disabled: false,
});

const onSubmit = () => {
  console.log("submit!", form);
};
// ---------------------------------------- Lifecycle & Computed ---------------------------------------- //
onMounted(() => {
  setTimeout(() => {
    form.file1 = `1220行程单.pdf||/pay/2023/12/07/11/203c564816424665b5bb006b08d82ef8.pdf||public`;
    form.file2 = `2.pdf||/pay/2023/12/11/15/3924cd818c9b4ea8a891f17ba1a9dc0f.pdf||public`;
  }, 1000);
});
// ---------------------------------------- Functions Business ------------------------------------------ //

// ---------------------------------------- Init -------------------------------------------------------- //
</script>

说明 ​

通常情况下，组件都是包含在表单中，因此针对查看操作时，若设置了el-form的disabled属性，则kd-upload默认时禁用删除操作，只能进行下载操作。

在其他场景下，你也可以通过showDelete和showDownload属性来自由控制是否显示该两项操作。
