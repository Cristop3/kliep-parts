Filter-table 过滤器+表格 ​

这是根据中后台列表页面常见搜索项、表格作整合，减少冗余代码的书写，一般搭配kd-page一起使用，也可以独立拆分只使用Filter或者只使用Table

组合Page-Filter-Table ​

插槽自定义控件如何编写 ​

  • formItemType:slot
  • filterXxx
  • modalValue,updateModelValue
js
1. formItemType设置为slot
2. 在组件内部使用name为filterXx的插槽，为了区分其他插槽，均以filter+字段首字母大写绑定
3. 需要绑定控件:model-value及change|input等等事件 相当于重写v-model语法糖 为啥不直接绑定v-model，因为走插槽上来的数据不是响应式的，因此只有重实现v-model，这样Filter内部才会获取到当前控件值，请确保执行上述操作

{
    formItemType: "slot",
    formItemLabel: "我是slot控件",
    formItemKey: "field",
},

<kd-filter-table>
    <template #filterField="{ modelValue, updateModelValue }">
    	<el-switch :model-value="modelValue.value" @change="updateModelValue" />
    </template>
</kd-filter-table>

Filter属性 ​

js
export type TFilterProps = {
  // filter
  filterName?: string; // 过滤器名称唯一标识 必传 不传时默认取当前router的name 当多组使用时 需要传入用于唯一标识
  filterList?: TFilterItem[]; // 过滤器表单集合
  filterCol?: {
    xs?: number;
    sm?: number;
    md?: number;
    lg?: number;
    xl?: number;
  }; // 自定义过滤器的布局 参看EMP layout
  filterFieldsColon?: boolean; // 是否搜索字段中文后面跟“：”
  filterQueryCache?: boolean; // 是否需要缓存过滤器值 TODO: 暂无使用
  filterDefaultBtn?: boolean; // 是否默认操作按钮 搜索 重置
  filterCache?: boolean; // 是否缓存当前针对filter的调整 TODO: 暂无实现
};

export type TFilterItem = {
  filterKey: string | number; // 唯一id
  // common
  formItemType?: string; // 表单项类型 目前支持input select datePicker
  formItemLabel?: string; // 表单项标题
  formItemRules?: string; // 表单项校验
  formItemHidden?: boolean | Function; // 表单项隐藏
  formItemKey?: string; // 表单项key
  formItemInitValue?: any;
  formItemRequired?: boolean; // 表单项 是否必填
  // 被用到的公共属性
  placeholder?: string;
  disabled?: boolean;
  // select业务属性
  selectOptions?: any[];
  selectKeyValueField?: { value: string; label: string | Function };
  // datePicker业务属性
  datePickerReallyKeys?: string; // 针对daterange处理分别绑定真实的key 用逗号分割 用于http query
  // 公共事件
  click?: Function;
  blur?: Function;
  focus?: Function;
  clear?: Function;
  change?: Function;
  input?: Function;
  visibleChange?: Function;
  // select特殊事件
  removeTag?: Function;
  // datePicker特殊事件
  calendarChange?: Function;
  panelChange?: Function;
};

Table属性 ​

  • 除开自定义的属性外，其余均走ElementPlus属性
js
TTableProp为官网默认属性

export interface ITableProps extends TTableProp {
  /** 唯一键，如果单个页面有多个表格实例，但是您只获取到一个表格实例，设置 `key` 即可解决，不过大多数情况下不需要设置，会自动处理 */
  tableKey?: String;
  /** `Table-column` 配置 `该属性为必填属性` */
  tableColumns?: Array<ITableColumns>;
  /** 对齐方式，默认值 `left` */
  tableAlignWhole?: Align;
  /** 表头对齐方式，若不设置该项，则使用表格的对齐方式 */
  tableHeaderAlign?: Align;
  /** 当内容过长被隐藏时显示 `tooltip`，默认值 `true` */
  tableShowOverflowTooltip?: boolean;
  /** 鼠标经过行时，行的背景色，默认 `--el-table-row-hover-bg-color`，具体看 https://github.com/element-plus/element-plus/blob/dev/packages/theme-chalk/src/table.scss#L602-L606 */
  // tableRowHoverBgColor?: string;
  /** 分页相关配置 */
  tablePagination?: IPaginationProps;

  tableEnableTools?: boolean; // 是否启用表格工具按钮
  tableSelectionNoAll?: boolean; // 是否selection模式下 无全选
  tableDataRenderAgain?: boolean; // 是否针对已获取的数据进行二次更新
  tableRequestMethod?: Function; // 获取表格请求url
  tableFilterMethod?: Function; //   增加前端分页下 筛选逻辑由调用方处理 返回应该的table list 这边只作slice分割显示
  tableAutoRowIndexSort?: boolean; // 是否让组件来进行序号排序
  tableData?: any[]; //   增加透传表格数据 不走Http获取
  tableStyle?: string; // 设置table style 比如--el-table-border-color: none
}

分页器属性 ​

TPaginationProp为官网默认属性

export interface IPaginationProps extends TPaginationProp {
  /** 自定义分页样式 */
  paginationStyle?: CSSProperties;
  /** 自定义类名 */
  paginationClass?: string;

  paginationCurrentPageKey?: string; // 当前页数 业务key
  paginationPageSizeKey?: string; // 每页显示条目个数 业务key
  paginationTotalKey?: string; // 总条目数 业务key
  paginationCurrentPage?: number; // 初始化 页数
  paginationPageSize?: number; // 初始化 每页显示个数
  paginationPosition?: Align; // 分页器位置
  paginationMb?: number | boolean; // 分页器margin-bottom数据
  paginationMt?: number | boolean; // 分页器margin-top数据
  paginationhidden?: false; // 增加前端数据源中 不采用分页显示 以最大量展示
}
